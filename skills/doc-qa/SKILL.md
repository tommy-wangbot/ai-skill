# 文稿快校

**原则：只说有问题的地方，没有问题不输出那个类别。**

## 执行步骤

1. 通读全文，边读边标记疑点
2. 对每个疑点判断：是否真的有问题？影响有多大？
3. 丢掉不重要的，保留值得说的
4. 按严重程度排序输出

## 问题分类（仅用于排序判断，不强制逐一呈现）

| 优先级 | 类型 | 典型问题 |
|--------|------|----------|
| 高 | 方向/逻辑 | 结论答非所问、推导断层、前提假设未说明、章节自相矛盾 |
| 中 | 结构/数据 | MECE 有漏洞、数字前后不一致、关键数据无来源 |
| 低 | 语言/格式 | 模糊用词、AI腔、标点混用、编号错乱 |

## 输出格式

```
【高优先】
① [位置] 问题描述 → 建议

【中优先】
① [位置] 问题描述 → 建议

【低优先】
① [位置] 问题描述 → 建议

总体：X/10 — 一句话判断（主要短板在哪）
```

**规则：**
- 某个优先级没有问题，整块不出现
- 每条控制在2行内
- 位置用章节名或段落首句关键词定位，不用页码
- 语言层问题直接给出：原文 → 建议改法
- 最多列 10 条，超出的合并或只取最重要的
