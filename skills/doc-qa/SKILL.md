# 咨询文稿校对（doc-qa）

对完成稿进行系统性质量审查。按以下顺序执行六个维度，由内到外逐层检查。

---

## 执行顺序

```
维度一：战略层  →  方向对不对
维度二：结构层  →  架构立不立
维度三：逻辑层  →  推导通不通
维度四：证据层  →  数据实不实
维度五：语言层  →  表达清不清
维度六：格式层  →  呈现齐不齐
```

先判断内容是否成立（维度一至四），再检查表达与呈现（维度五至六）。

---

## 维度一：战略层（Directional）

检查文稿是否解决了正确的问题、面向正确的受众、划清了正确的边界。

- **需求匹配度**：结论是否直接回答客户/业务方的核心诉求？有无答非所问？
- **受众同理心**：术语是否符合读者认知水平？关键结论是否在前3页出现？
- **边界完整性**：是否明确界定"做什么"与"不做什么"（In-scope / Out-of-scope）？

---

## 维度二：结构层（Structural）

检查框架是否符合金字塔原理，分类是否 MECE，叙事线是否清晰。

- **横向 MECE**：并列对策/原因是否有交叉重叠？是否有重大遗漏？
- **纵向层次**：各级标题层次是否一致？子项是否从属于其上级？
- **叙事主线**：是否遵循"问题 → 分析 → 洞察 → 建议"结构？有无章节游离于主线？
- **模块归属**：是否有内容放错了章节位置？

---

## 维度三：逻辑层（Logical）

检查论证链条是否完整，因果关系是否成立，结论是否能从分析推出。

- **纵向关联**：结论有论点支撑？论点有数据/事实支撑？（Why-So / So-What 双向检验）
- **因果闭环**：解决方案是否能推导出预期收益？是否存在逻辑断层？
- **假设显性化**：关键推导的前提假设是否已明示，而非隐含？
- **逻辑谬误**：是否存在无根据跳跃、相关性当因果、循环论证？

---

## 维度四：证据层（Evidence）

检查数据、事实、引用的内部一致性与可信度。

- **数字一致性**：同一数据在不同章节是否前后一致？单位是否统一？
- **来源标注**：关键数据是否有来源？是否存在悬空主张（有结论无依据）？
- **时效性**：引用数据或假设是否过时？时间表述是否有冲突？
- **章节矛盾**：不同章节的事实陈述是否相互矛盾？

---

## 维度五：语言层（Language）

检查措辞是否准确、专业、简洁，无 AI 腔，无歧义。

- **措辞精准**：是否有模糊表述（"较大"、"相关"、"一定程度上"）未给具体量化？
- **术语一致**：同一概念全文是否使用同一术语，无混用？
- **句子质量**：是否有过长句（超50字）、冗余词语、填充词（"众所周知"、"值得注意的是"）？
- **风格适配**：语言是否符合咨询报告风格？是否有明显 AI 生成腔需要去除？

---

## 维度六：格式层（Format）

检查版面格式的一致性与规范性。

- **编号规范**：标题编号、列表编号是否连续、层次一致？
- **标点统一**：中英文标点是否混用？句末标点是否缺失？
- **大小写/全半角**：英文缩写大小写、数字全半角是否统一？
- **图表标注**：图表是否有标题/来源？正文引用编号是否对应？

---

## 输出格式

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
校对报告 | [文稿标题] | [日期]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【维度一：战略层】
① ...
② ...
（无问题则写"✓ 无异常"）

【维度二：结构层】
① ...

【维度三：逻辑层】
① ...

【维度四：证据层】
① ...

【维度五：语言层】
① 原文："..." → 建议："..."

【维度六：格式层】
① ...

【关键风险提示】
（列出影响整体交付质量的高优先级问题，需优先处理）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

每个问题标注位置（章节/段落），格式为：`位置 + 问题描述 + 建议修改`。
